## What This Is

A Pulumi Terraform Bridge provider that wraps the [Nutanix Terraform provider](https://github.com/nutanix/terraform-provider-nutanix), exposing it as a Pulumi provider for JavaScript/TypeScript, Python, Go, and .NET.

## Common Commands

```bash
make development      # Full dev build: install plugins → provider → lint → SDKs → install SDKs
make provider         # Compile the provider binary only
make tfgen            # Regenerate schema.json from the upstream Terraform provider schema
make build_sdks       # Regenerate all language SDKs from schema.json
make lint_provider    # Run golangci-lint on provider Go code
make test             # Integration tests in examples/ (requires live Nutanix environment)
make clean            # Remove generated SDK output
```

Building individual SDKs: `make build_nodejs`, `make build_python`, `make build_go`, `make build_dotnet`.

## Architecture

### Provider layer (`provider/`)

- `resources.go` — the only file that typically needs editing for resource/datasource changes. It maps Terraform resource names (`nutanix_<x>`) to Pulumi tokens (`nutanix:index:<X>`) and configures the `tfbridge.ProviderInfo`.
- `cmd/pulumi-tfgen-nutanix/main.go` — the schema generator binary; reads `resources.go` and emits `schema.json`.
- `cmd/pulumi-resource-nutanix/main.go` — the runtime provider binary served by the Pulumi engine.

### Schema (`provider/cmd/pulumi-resource-nutanix/schema.json`)

Auto-generated from `resources.go` by `make tfgen`. Do not edit by hand — changes will be overwritten.

### Generated SDKs (`sdk/`)

All files under `sdk/nodejs/`, `sdk/python/`, `sdk/go/`, and `sdk/dotnet/` are generated by `make build_sdks`. Commit the generated output after any schema change.

### Examples / integration tests (`examples/`)

`examples_test.go` drives integration tests using `github.com/pulumi/pulumi/pkg/v3/testing/integration`. Tests require a live Nutanix environment configured via environment variables. Run with `make test` (parallel=4, 2h timeout).

## Development Flow

1. Edit `provider/resources.go` to add or modify resource/datasource mappings.
2. Run `make tfgen` to regenerate `schema.json`.
3. Run `make build_sdks` to regenerate all language SDKs.
4. Commit the generated schema and SDK files together with the provider change.
5. Run `make lint_provider` to check Go code.

## CI

- `main.yml` — builds provider and SDKs on push to `main` (matrix: Go 1.24.x, Node 20.x, Python 3.11, .NET 8.0.x).
- `run-acceptance-tests.yml` — runs integration tests; triggered manually on approved PRs.
- `release.yml` — publishes to npm, PyPI, and NuGet on tag push; binaries via GoReleaser.
